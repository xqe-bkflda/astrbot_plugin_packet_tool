# 嘿嘿～ QQ协议包工具插件 (astrbot_plugin_packet_tool)

一个专门给 AstrBot 用的 QQ 协议包小工具，能帮你翻聊天记录、抓链接、发/看 JSON 卡片、找黑条消息、向上考古、自定义发包，甚至还能配置签名服务器（虽然可能没什么卵用 😅）。

## 作者

- GitHub：[xqe-bkflda](https://github.com/xqe-bkflda)

## 功能一览

| 命令 | 说明 | 例子 |
|------|------|------|
| `/取信息` 或 `/抓包` | 获取被回复消息的原始 JSON，以转发卡片形式返回 | 回复一条消息，发送 `/取信息` |
| `/取链接` | 提取被回复消息中的所有链接 | 回复一条消息，发送 `/取链接` |
| `/发卡 <JSON>` | 发送一张 JSON 卡片（会自动签名） | `/发卡 {"app":"com.tencent.structmsg","desc":"..."}` |
| `/看卡` | 解析被回复消息或上一条消息中的 JSON 卡片并展示 | 回复一条卡片消息，发送 `/看卡` |
| `/发小程序 <JSON>` | 发送小程序卡片（同上，也是发 JSON） | `/发小程序 {"app":"xxx"}` |
| `/取黑条消息` | 向上查找最近一条被风控/警告/撤回的系统通知或黑条消息（仅群聊） | 直接发 `/取黑条消息` |
| `/向上获取 <n>` | 获取往上数第 n 条消息的原始 JSON（默认 n=1，即上一条） | `/向上获取 3` |
| `/发包 <JSON>` | 自定义发送任意原始 JSON 协议包（会签名） | `/发包 {"action":"send_msg","params":{...}}` |
| `/配置签名服务器 <url>` 或 `/setsign` | 修改或清空签名服务器地址（管理员专用） | `/配置签名服务器 http://127.0.0.1:8080/sign` |

## 签名服务器（重要提醒 ⚠️）

插件默认使用 `http://h2.secnet.cc:26321/sign` 作为签名服务器。  
但是！**这个配置功能大概率是没用的** —— 因为 AstrBot 底层可能已经自己处理过签名了，或者签名服务器根本不需要调。  
如果你遇到发卡/发包失败，请先检查 AstrBot 自身是否支持签名，而不是折腾这个设置。当然，留着它也不碍事，你可以当它是个摆设 😇

## 数据存储

所有配置（如自定义的签名服务器地址）都会保存在 AstrBot 数据目录下：  
`data/plugin_data/astrbot_plugin_packet_tool/config.json`  

消息缓存只存在内存里（最多保留 100 条），重启机器人就清零，不用担心隐私问题。

## 依赖

- `aiohttp`（一般 AstrBot 自带，如果没有请手动装一下）
- Python 3.8+

## 安装方法

1. 将本插件文件夹放到 AstrBot 的 `addons` 目录下，文件夹名建议为 `astrbot_plugin_packet_tool`。
2. 重启 AstrBot，或在 WebUI 中启用插件。
3. 在群里或私聊测试 `/取信息` 即可。

## 小提示 💡

- `/取黑条消息` 依赖于消息缓存，刚启动机器人时历史不够多会失败，聊一会就有啦。
- `/向上获取` 的数字别太大，缓存只有 100 条。
- 回复消息时使用 `/取链接`、`/取信息` 等命令，插件会自动识别你回复的那一条。

## 遇到问题？

先去检查一下 AstrBot 日志，大概率是签名没通过或者 JSON 格式写错了。  
如果还是不行…… 那就当这个插件是个可爱的玩具吧 🧸
